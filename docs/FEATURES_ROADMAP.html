Product Features
DocSera App Patient Application January 2026

1. Application Overview

DocSera is a mobile healthcare application that enables patients in Syria to discover medical practitioners, book appointments, manage their health records, and communicate securely with doctors. The application provides a comprehensive healthcare management experience with full support for Arabic (RTL layout) and English languages.

Platforms: iOS and Android
Languages: Arabic (RTL) and English

3. Doctor Discovery & Search

Search & Filters
Advanced doctor discovery system
  • Text Search — Search by doctor name, clinic name, or specialty. Results show matching doctors and specialty suggestions (selecting a specialty leads to location selection)
  • Step-Based Search — When not using direct text search: select specialty first, then choose location (city or nearby)
    • Specialty Selection — localized medical specialty names
    • City Selection — choose from predefined city list
    • Nearby Option — uses device GPS to find doctors within radius
  • Gender Filter — Filter results by male/female practitioners
  • Favorites Section — Quick access to saved favorite doctors
  • Map View — Interactive map showing doctor locations with pins
Doctor Profile
Complete practitioner information display
  • Basic Information — Name, title, specialty, gender, clinic name
  • Profile Description — Rich text bio with multiple sections (QuillJS format)
  • Offered Services — List of services with optional descriptions
  • Location
    • Clinic name, street address, building number
    • City and country
    • Additional location details
    • "Open in Maps" button with coordinates
  • Gallery — Photo gallery with full-screen viewer, zoom, swipe
  • Languages Spoken — Arabic, English, French, German, Turkish, etc.
  • Contact Options — Phone (tap to call), Email (tap to compose)
  • Favorites — Add/remove from favorites list
  • Share — Share profile via system share sheet + QR code

  • Patient Selection — Book for self or any registered relative
  • Visit History — Indicate if first visit or follow-up
  • Visit Reason — Select from doctor-defined consultation types
  • Time Slot Selection — Calendar with available slots from doctor's schedule
  • Confirmation — Review details before submitting
  • Booking Result
    • If doctor requires manual approval: status page indicating appointment is pending confirmation
    • If doctor has auto-accept enabled: confirmation page showing appointment is booked successfully

Appointment Management

  • Tab View — Toggle between "Upcoming" and "Past" appointments
  • Appointment Card — Doctor info, date/time, status badge
  • Status Tracking — Pending, Confirmed, Completed, Cancelled, Rejected
  • Appointment Details — Full details page with all booking information
  • Cancel Appointment — Cancel with confirmation dialog. Available only within the doctor-defined time limit before the appointment; otherwise an unavailability notice is displayed
  • Reschedule Request — Request a new time slot. Available only within the doctor-defined time limit. If doctor requires approval, the request must be confirmed before the change takes effect
  • Visit Reports — View reports attached to completed appointments
  • Load More — Pagination for large appointment history
  • Tab Persistence — Remembers last selected tab (SharedPreferences)

Comprehensive personal medical history management. Each record type supports full CRUD operations and can be managed separately for the main user and each registered relative.

Common Record Features

Each health category has specific input fields tailored to its type. Common features across categories include:

  • Add Record — Bottom sheet form with searchable master data
  • Category-Specific Fields — Each category has unique inputs (e.g., severity for allergies, family member relationship for family history, dosage for medications)
  • Confirmed Status — Mark as doctor-confirmed or self-reported
  • Source Tracking — Records whether added by patient or doctor
  • View Details — Dialog showing all record information
  • Delete Record — Confirmation dialog before deletion
  • "No Items" Declaration — Explicitly declare "No allergies" etc.
  • Patient Switching — Switch between self and relatives, records update automatically

Visit Reports

Doctor-generated reports from completed appointments, accessible in a dedicated section:

  • Reports List — All reports from completed appointments displayed with doctor info and date
  • Year/Month Filters — Filter reports by year and month for easy navigation
  • Search — Search reports by doctor name
  • Report Details — View full report with diagnosis, recommendations, and clinic information
  • Print Report — Generate and print PDF version of the report
  • Share Report — Share report as PDF via system share sheet

Documents & Notes

Personal storage for medical documents, accessible only by the patient (doctors do not have access):

  • Document Upload — Upload images and PDF files for personal reference
  • Document Types — Lab results, prescriptions, imaging, other
  • Document Preview — In-app preview with zoom
  • Personal Notes — Rich text notes with QuillJS editor
  • Private Storage — All documents and notes are for personal use only

6. Secure Messaging

Conversation Features
Direct patient-doctor communication
  • Message Request — First message initiates a request; patient waits for doctor to accept and open the conversation
  • Text Messages — Standard text messaging with auto-scroll
  • Voice Messages — Record and send M4A audio with duration display
  • Image Attachments — Select up to 8 images, compression before upload
  • PDF Attachments — Share PDF documents in chat
  • Real-time Delivery — Messages delivered via Supabase Realtime
  • Optimistic Updates — Messages appear immediately, sync in background
  • Retry Failed Messages — Retry button for failed sends
  • Image Viewer — Full-screen overlay with zoom, swipe navigation
  • Conversation States
    • "Waiting for doctor reply" banner (before first doctor response)
    • "Conversation closed" banner (doctor closed the conversation; patient cannot send new messages)
    • If blocked by doctor: appears as closed, and patient cannot initiate new conversations with this doctor (for self or relatives)
  • Doctor Control — Doctor has full control over conversation: can accept, close, or block at any time
  • Relative Conversations — Banner shows when messaging on behalf of relative

Messages List

  • Grouped by Doctor — Conversations grouped by practitioner
  • Year Headers — Organized by year for easy navigation
  • Unread Indicator — Visual indicator for unread messages
  • Last Message Preview — Shows latest message snippet

7. Account & Security

Personal Information

  • My Profile — Edit name, birthday, gender, and address (optional)
  • My Relatives — Add/edit/remove family members

Login Credentials

  • Phone Number — View/change with OTP verification
  • Email — View/change with OTP verification
  • Password — Change password (current + new)

Security Settings

  • Two-Factor Authentication — Enable/disable 2FA for login
  • Encrypted Documents — Info about document encryption (always on)
  • Biometric Login — Face ID / Touch ID toggle with device detection
  • Trusted Devices — Email OTP required for login from new devices

Preferences & Privacy

  • Language — Switch between Arabic and English
  • Legal Information — Terms and Conditions of Use, Personal Data Protection Policy, Report Illicit Content
  • Deactivate Account — Disable account with confirmation (account is deactivated, not permanently deleted)
  • Logout — Sign out of the application

Additional Account Features

  • Privacy Info Card — Information card displaying data protection and privacy assurance
  • App Version — Version number displayed at bottom

8. Authentication

  • Email Registration — Sign up with email address and password
  • Email/Password Login — Primary authentication method
  • Phone Input Support — Users can enter phone number at login (system maps to associated email)
  • Email OTP Verification — OTP codes sent to email for: email verification, and two-factor authentication
  • Two-Factor Authentication — When enabled, OTP is sent to email after entering credentials
  • Biometric Login — Face ID / Touch ID after initial email/password setup
  • New Device Detection — Email OTP required when logging in from untrusted devices
  • Terms of Use — Must accept before registration

9. Push Notifications

  • Appointment Reminders — Notifications for upcoming appointments
  • Status Updates — When appointment is confirmed/rejected/cancelled
  • New Messages — Notification when doctor sends message
  • Visit Reports — Notification when doctor adds/updates report
  • Deep Links — Tap notification to navigate directly to relevant screen

10. Technical Highlights

Framework Flutter (Dart) — Single codebase for iOS and Android
State Management BLoC/Cubit pattern throughout the application
Backend Supabase (PostgreSQL, Auth, Realtime, Storage, Edge Functions)
Real-time Supabase Realtime for messages and appointment updates
Security Row-Level Security (RLS) on all patient data tables
Localization Full RTL support with flutter_localizations
Biometrics local_auth + flutter_secure_storage
Push Notifications Pushy service for cross-platform notifications
